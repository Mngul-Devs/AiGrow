Local Processing

Python Script / Server (running YOLO)

Continuously pulls the Dahua camera's RTSP stream and runs YOLO inference on each frame.

The system operates in real-time, processing video feeds 24/7 to detect objects of interest with minimal latency.

2

Event Trigger

Python Script

When the YOLO model detects your target object, the script is triggered. It captures the full frame and/or the cropped snapshot.

Detection thresholds can be customized based on confidence scores to minimize false positives and ensure accurate plant health assessment.

3

Data Structuring

Python Script

It compiles the event metadata into a structured format (usually JSON):

  • Timestamp - Exact date and time of detection
  • Camera ID - Unique identifier for multi-camera setups
  • Object Class - Type of plant or issue detected
  • Confidence Score - Model certainty percentage
  • Bounding Box Coordinates - Precise location in frame
4

Pushing Data

Python Script (Database Connector)

1. Metadata: The script uses a database connector (e.g., psycopg2 for PostgreSQL, mysql.connector for MySQL, or the sqlite3 library for a local file-based DB) to INSERT the JSON metadata into a table.

2. Snapshot File: It uploads the saved snapshot JPEG/PNG file to your server's file storage (like an S3 bucket, a mounted NAS, or a simple directory) and stores the resulting file path or URL in the same database record.

This dual approach ensures both quick metadata queries and efficient image retrieval for analysis and reporting.

üîÑ n8n Workflow

Automated Workflow Integration

1

Local Processing

Python Script / Server (running YOLO)

Same as above: pulls RTSP, runs inference, and captures the snapshot.

The system continuously monitors the camera feed and processes each frame through the YOLO model to detect target objects in real-time.

2

n8n Setup

n8n Workflow

You start a new n8n workflow with a Webhook Trigger node. This node generates a unique URL that waits for incoming data.

The webhook acts as the entry point for your automation, allowing external systems to trigger complex workflows with a simple HTTP request.

3

Triggering n8n

Python Script (HTTP Request)

When YOLO detects an object, the script performs an HTTP POST request to the n8n Webhook URL.

  • The JSON metadata is sent in the request body
  • The snapshot image data is sent as a Base64 string
  • Headers include content-type and authentication tokens

This approach ensures secure, reliable data transmission between your detection system and automation workflow.

4

Workflow Execution

n8n Workflow

The Webhook node receives the data, which triggers the rest of the workflow:

  • 1. File Write Node: Decodes the Base64 image and saves the snapshot to your file storage (local directory, S3, or NAS).
  • 2. Database Node: Inserts the metadata into your server database (PostgreSQL, MySQL, MongoDB).
  • 3. Notification Node: Sends an alert (e.g., to Telegram, Slack, or Email) with the snapshot attached and the metadata in the message.

All three operations run in parallel for maximum efficiency, ensuring instant notifications while maintaining data integrity.
